To be accepted by F-Droid, the app must meet the following criteria:

- [x] **No Binary Blobs**: All libraries must be built from source. (F-Droid's build server handles this).
- [x] **No Self-Updaters**: The internal GitHub updater must be disabled for the `fdroid` flavor.
    - *Status*: Fixed in `src/utils/githubUpdater.ts` using `IS_FDROID_BUILD` flag.
- [x] **Free Software License**: The project must be under a GPL, Apache, or MIT-compatible license.
- [ ] **Anti-Features Declaration**: Any non-free dependencies or network services must be declared.

- [x] **Reproducible Builds**: (Optional but recommended) Ensure the build is deterministic.
