# Release Notes - v2.19.9

## FlashList Stale Render Resolution

This release addresses the persistent issue with note multi-selection where entering selection mode did not update the header selection actions bar, and tapping other notes resulted in navigating to their editor page instead of toggling selection.

### Key Changes
- **ListHeader Memoization**: Converted `ListHeader` into a `useCallback`-memoized functional component `renderHeader`. This ensures the header element correctly updates its internal layouts when `isSelectionMode` or `selectedIds` state transitions.
- **RenderItem Memoization**: Wrapped the list item renderer in a `useCallback` hook (`renderItem`). This binds the click event handlers and selection state to explicit rendering dependencies, preventing cells from using stale closures.
- **FlashList Syncing**: Configured `extraData` to pass an object referencing the current selection state fields so FlashList's view recycling engine correctly schedules diffing operations for all visible cells.
