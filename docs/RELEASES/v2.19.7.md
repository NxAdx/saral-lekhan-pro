# Release Notes - v2.19.7

Date: 2026-06-26
Tag: `v2.19.7`

This release resolves the race conditions and state sync bugs that could cause selection mode to reset prematurely while keeping note items visually checked.

## What changed

### 1) Decoupled Selection Lifecycle
- **Removed Stale Prune Watcher:** Discarded the background `useEffect` that was checking note IDs against `filteredNotes`. This hook could prune active selection states prematurely during intermediate renders where list filter computations were still running, causing the header and item handlers to drop selection state while leaving the card tick visually checked.
- **Event-Driven Selection Clear:** Added clean event-based clear listeners that automatically trigger only when the user explicitly changes the filter tag or enters a new search query.
- **Explicit List Refresh Tracking:** Upgraded the `extraData` dependency string of `FlashList` to incorporate the selection mode state, size, and selected elements in a single string, ensuring complete sync on all viewports.

### 2) Direct OTA Updater Track
- Bumped app build targets to `2.19.7` (versionCode `108`).
