# Walkthrough - v2.8.1 Stability & UI Reversion

This release focuses on stability, fixing a fatal crash, and reverting UI placement of support features according to user feedback.

## Key Fixes

### 🛡️ Fatal Crash Resolution
- **Issue**: App would crash with `TypeError: Cannot read property 'title' of undefined` during note deletion.
- **Fix**: Added a robust null guard in `BentoCard.tsx`. If a note is removed from the state but a re-render is triggered before the list updates, the component now safely returns `null` instead of crashing.

### ✍️ Editor Logic Improvements
- **Quote Breakout**: Fixed an issue where users could infinitely nest blockquotes.
- **Improved Script**: Pressing **Enter** on an empty line inside a `BLOCKQUOTE` now outdents the text, effectively exiting the quote block.
- **Typo Fix**: Corrected a library-specific typo `injectJavascript` in `[id].tsx` to ensure predictable script injection.

### 🎨 UI Contrast & Reversion
- **Settings Screen**: "Report Bug" and "History" (Changelog) have been moved back to the **Settings** screen in a dedicated "Support & Updates" section.
- **Home Screen**: Removed the redundant buttons and modals from the Home header to keep the interface clean for note-taking.
- **Modal Visibility**: Forced high-contrast labels (White in Dark Mode) in `ThemedModal.tsx` to address visibility issues in the export/share menu.
