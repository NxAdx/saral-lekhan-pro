# Release Notes - v2.9.4

This release focuses on production reliability and polish across authentication, startup visuals, and updater UX.

## Key Fixes

### 1) Google Sign-In `DEVELOPER_ERROR` hardening
- Fixed Android release signing path so `release` builds no longer default to debug keystore.
- Added explicit release signing wiring via `MYAPP_UPLOAD_*` env vars in `android/app/build.gradle`.
- Kept OAuth Web Client flow (`webClientId`) and improved runtime error mapping for clearer diagnostics.
- Hardened CI to validate `GOOGLE_SERVICES_JSON` structure before build.

### 2) Startup white-screen reduction
- Unified startup background to `#171513` in:
  - `app.json` splash config
  - `android/app/src/main/res/values/colors.xml`
  - `android/app/src/main/res/values-night/colors.xml`
- Added `android:windowBackground` to `AppTheme` in `styles.xml`.
- Added early `SystemUI.setBackgroundColorAsync('#171513')` in root layout bootstrap.

### 3) Updater card/button polish
- Applied stable green-border visual treatment for updater card and "Check for Updates" button.
- Removed temporary hardcoded `HOTFIX` app version override and centralized version source.
- Replaced naive version mismatch check with semantic-part comparison to avoid false positives.
