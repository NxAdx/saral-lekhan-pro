### 3) Build and CI hardening
- CI now fails fast if release signing secrets are missing (`20739df`).
- Workflow keeps `permissions: contents: write` for release upload path (`18099c0`).
- `GOOGLE_SERVICES_JSON` validation remains enforced in CI before Gradle build.

## Operational notes (important)

### GitHub Release publish requirement
If Action logs show:
- `Resource not accessible by integration`
- `GITHUB_TOKEN Permissions: Contents: read`

Then GitHub release upload will fail, and in-app updater will not receive fresh release metadata.

Fix in repo settings:
1. `GitHub -> Settings -> Actions -> General`
2. `Workflow permissions -> Read and write permissions`
3. Save.
4. Re-run the tagged workflow.

### Firebase / OAuth requirement
If app shows `Google Sign-In config mismatch for package com.sarallekhan`, the installed APK signature still does not match Firebase fingerprints for this Android app. Re-register SHA-1/SHA-256 and refresh `google-services.json` in both local build and CI secret.
