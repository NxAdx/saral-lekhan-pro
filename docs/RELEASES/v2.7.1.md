# Reliability & Support Updates (v2.7.1)

This update focuses on app stability and providing tools for better debugging and support.

## 🛡️ Stability Fixes: Editor Crash Resolved
I have identified and fixed the crash that occurred when re-opening the same note.
- **Nested Scrolling Fix**: Resolved a conflict between the outer `ScrollView` and the `RichEditor` internal container.
- **Instance Management**: Added a unique `key` to the editor to ensure a fresh instance on every entry.

## 🛠️ Bug Reporting & Crash Resilience
The bug tracking system is now robust enough to survive app crashes.
- **Persistent Storage**: Logs are now saved to the device's storage (`app_logs.json`) in real-time. This means that if the app crashes, the logs from that session are still available when you reopen the app.
- **Auto-Logging Crashes**: I've implemented a global error handler. If a fatal JavaScript error occurs, the app will automatically log the error message and stack trace before closing, allowing you to share it via the "Report Bug" feature.
- **Early Initialization**: The logging system starts as soon as the app launches, capturing even the earliest events.

## 📍 Final Placement
- **Settings Screen**: The **"Report Bug"** button has been moved to the very bottom of the Settings screen (the final item), ensuring it is always accessible when you need support.

## 🚀 How to Verify
1.  **Test Stability**: Open a note, go back, and open it again. Repeat multiple times. The app should remain stable and responsive.
2.  **Test Bug Reporting**:
    - Go to **Settings**.
    - Scroll down to the new **Support** section.
    - Tap **Report Bug**.
    - Verify that the app generates a `.txt` file with debug info and logs, and opens the sharing dialog.

> [!TIP]
> Providing debug logs in your reports helps us identify the exact cause of issues on specific devices, leading to much faster fixes.
