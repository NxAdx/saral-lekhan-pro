# Release Notes - v2.19.10

## Selection State Mapped Rendering & Visibility Fixes

This release resolves two critical bugs in the selection mode functionality:
1. Recycled cell data lagging (closures retaining `isSelectionMode = false`) and static header lagging.
2. Selection rings (empty circles for unselected items) being completely invisible in dark themes.

### Key Changes
- **Selection State Mapping in Data**: Instead of reading selection states from the parent closure inside cell renderers, the selection state is mapped directly onto the elements of the list `data` array:
  ```typescript
  const dataToRender = useMemo(() => {
    return filteredNotes.map(note => ({
      ...note,
      isSelected: selectedIds.has(note.id),
      isSelectionMode,
    }));
  }, [filteredNotes, selectedIds, isSelectionMode]);
  ```
  Since `data` elements change when selection state transitions, Shopify's `FlashList` diffing engine is forced to call `renderItem` for all visible cells. The cell renderers read their properties directly from the `item` parameter, completely resolving stale closure lag.
- **Header Refresh**: Restored `ListHeader` back to a static JSX element wrapper. Since the list re-renders on selection data changes, FlatList/FlashList correctly updates the header component dynamically, preserving focus on the search bar.
- **Selection Circle Contrast**: Changed the outline border of `unselectedIcon` in `BentoCard` from `colors.strokeDim` to `colors.stroke`. In dark themes (like classic dark), `strokeDim` and card background (`bgRaised`) carry the exact same color value (`#2B2926`), rendering the selection outlines invisible. Using `colors.stroke` ensures high visibility in all themes.
