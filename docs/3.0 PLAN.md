# Saral Lekhan v3.0.0 Production Phase Plan

## Summary
This phase should produce a **production launch candidate**, not another patch release. The work is organized to protect the current stable build while moving the app toward Play launch, monetization, better UX, and better performance.

The phase is:

1. **Platform and Play readiness**
2. **Launch-path and startup hardening**
3. **Branding and typography normalization**
4. **Monetization foundation**
5. **Feature wave 1**
6. **UX/system consistency pass**
7. **Release safety and rollback discipline**

This plan keeps **`v2.16.7` as the rollback anchor** and assumes:
- production-first scope
- phased upgrade strategy
- Android-first widgets
- incremental editor evolution
- free + Pro model
- Play + direct distribution
- monetization foundation now, activation after testing
- annual + lifetime Pro plans

## Key Changes

### 1. Platform and Play Readiness
- Upgrade from **Expo SDK 49 / RN 0.72** to a **Play-safe Expo baseline** that supports **API 35**. Recommended minimum: **Expo SDK 53**.
- Move Android build targets to:
  - `compileSdkVersion 35`
  - `targetSdkVersion 35`
- Split Android distribution into two explicit channels:
  - `play`: AAB-first, Play-safe, no GitHub APK updater flow
  - `direct`: APK/sideload build, GitHub updater retained
- Remove or channel-gate risky permissions:
  - remove from Play build: `REQUEST_INSTALL_PACKAGES`
  - remove unless feature usage proves necessary: `READ_EXTERNAL_STORAGE`, `WRITE_EXTERNAL_STORAGE`, `RECORD_AUDIO`, `SYSTEM_ALERT_WINDOW`
- Align workflows/build profiles so Play and direct releases are built intentionally, not from one mixed path.

### 2. Launch Path and Startup Speed
- **Direct Launch Authority**:
  - Completely removed all splash screen logic (native and JS) to ensure immediate app launch.
  - App launches directly into the home UI or lock screen without intermediate branded phases.
  - Native splash is bypassed or set to minimum duration (1ms black/empty) to reduce production build complexity and speed up startup.
- Reduce startup-critical work:
  - DB init stays on critical path
  - AI/auth init move off the first-frame path unless required
  - runtime flags stay non-blocking
- Reduce font boot cost:
  - load only core app fonts for first frame
  - defer non-critical font assets
- Improve note/search performance:
  - add indexes for main note retrieval paths
  - stop repeated rich-text stripping on every search/render
  - search on cached plain text
  - debounce search input
- Normalize editor image handling so note content is stored consistently and remains lighter for backup/sync.

### 3. Branding and Typography
- Normalize installed/public app branding to **`Saral Lekhan`**.
- Home header primary title becomes exactly **`Saral लेखन`**.
- Replace credits text:
  - from `Designed & Developed by`
  - to **`Designed by`**
  - developer name becomes exactly **`Aadarsh Lokhande`**
- Restrict supported runtime font families to:
  - `Hind`
  - `Poppins`
  - `Noto Sans`
  - `Baloo 2`
- Remove these from app-font selection and normal UI typography:
  - `Yantramanav`
  - `Tiro Devanagari`
  - legacy decorative display/mono dependency in regular app UI
- Implement semantic typography tokens everywhere so switching fonts does **not** change perceived size unpredictably.
- Keep Devanagari-safe rendering rules:
  - proper line heights
  - `includeFontPadding: false`
  - stable fallback behavior where needed for Hindi/Marathi
- Keep the home title styling **brand-controlled**, not user-font-controlled, so `Saral लेखन` always looks intentional and stable.

### 4. Monetization Foundation
- Implement monetization plumbing now, but activate public selling after testing.
- Use **RevenueCat** over raw Play Billing-only state management.
- Plan structure:
  - `annual`
  - `lifetime`
- Free core remains strong:
  - local writing
  - local notes/search/tags/trash
  - checklist editing
  - import/export basics
  - biometric lock
- Pro surfaces in this phase:
  - Spark AI
  - Google Drive sync/backup
  - native print
  - premium themes/fonts
  - advanced editor blocks/template packs
  - advanced widget set
- Add paywall entry points in:
  - settings
  - Spark AI
  - Drive sync
  - print
  - premium appearance
  - advanced writing tools

### 5. Feature Wave 1
- **Checklist support**
  - add task-list editing to the current editor architecture
  - must persist through save, reopen, export/import, and sync
- **Native Print instead of Export to PDF**
  - replace the user-facing `Export to PDF` option with **`Print`**
  - use platform print flow on Android/iOS
  - if PDF generation remains needed internally, keep it hidden from the user
  - gate print as a **Pro feature**
- **Android Widget v1**
  - Android-first
  - Quick Capture
  - supports:
    - new note
    - new checklist
- **One extra monetizable feature**
  - recommended: **Template Gallery**
  - free starter templates
  - Pro template packs
- **Incremental editor upgrades after checklist**
  - divider
  - callout
  - quote
  - code block
  - slash insert
- Explicitly defer:
  - block-editor rewrite
  - tables
  - toggles
  - databases

### 6. UX and UI Consistency Pass
- Keep the visual language, but tighten it with stricter rules:
  - consistent header height
  - consistent icon-button sizing
  - better spacing rhythm
  - cleaner card density
  - more stable search layout
- Prioritize UX cleanup on:
  - home
  - editor
  - settings/paywall/update areas
- Break oversized settings logic into clearer sections/components.
- Make the home header feel more premium and calm, centered around `Saral लेखन`.

### 7. Release Safety and Rollback
- Freeze **`v2.16.7`** as the last known stable rollback baseline.
- Implement on a dedicated release branch with milestone tags:
  - `v3.0.0-rc1` platform/channel split
  - `v3.0.0-rc2` splash/startup/branding/typography
  - `v3.0.0-rc3` monetization foundation
  - `v3.0.0-rc4` feature wave 1
- Keep runtime-flagged rollback for JS UX where possible.
- Keep native splash/launch changes isolated for one-step revert if needed.
- Require GitHub release builds at each milestone before proceeding.

## Test Plan
- **Launch**
  - cold launch on Android 11/12/13/14/15
  - confirm one branded splash perception only
  - confirm no crash, no white flash, no second branded phase
- **Typography**
  - switch between all four supported fonts
  - verify visual size parity
  - verify Hindi/Marathi rendering stability
  - verify home title `Saral लेखन` remains visually stable and does not inherit broken user font sizing
- **Branding**
  - installed label shows `Saral Lekhan`
  - settings/about text shows `Designed by Aadarsh Lokhande`
- **Print**
  - print opens native print sheet on Android/iOS
  - layout preserves note formatting
  - Pro gate works correctly
- **Editor**
  - checklist create/edit/save/reopen/export/import/sync
- **Performance**
  - startup benchmark before/after
  - search responsiveness on large note sets
  - editor open/save timing with heavy notes
- **Distribution**
  - direct build keeps GitHub updater
  - Play build removes it
- **Monetization**
  - annual purchase
  - lifetime purchase
  - restore purchases
  - gates work for print, AI, sync, premium appearance
- **Release**
  - GitHub Actions green for Play and direct lanes
  - rollback path verified from stable baseline

## Assumptions
- `v2.16.7` remains the stable rollback anchor until `v3.0.0` is accepted.
- Home title should be exactly **`Saral लेखन`**.
- Installed app label stays **`Saral Lekhan`**.
- Print replaces the user-facing PDF export option.
- Only four font families remain user-selectable.
- Widgets are Android-first in this phase.
- Monetization plumbing ships now; sales can be activated after testing confidence is high.
