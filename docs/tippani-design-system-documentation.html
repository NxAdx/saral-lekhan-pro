01 — Overview & Design Principles

[cite_start]

Tippani (टिप्पणी) is a bilingual notes application built for Hindi-speaking users who think and write fluidly across Devanagari and Latin scripts[cite: 236]. The design system is anchored to a single powerful concept: the physical act of pressing a key.

1.1 The Three Pillars

  • Tactile First: Every interactive element must feel physical. Buttons press down (translateY +2px, shadow collapse) on tap. [cite_start]Users must feel they are pressing real objects[cite: 241].
  • Warmth Over Sterility: The palette is warm grey, never cool grey or clinical white. [cite_start]The single accent is burnt terracotta[cite: 244].
  • Bilingual by Design: Hindi (Devanagari) is not an afterthought. [cite_start]Every token, spacing rule, and font choice works equally well in both scripts[cite: 247].

02 — Color System

The palette has three functional layers: neutral surfaces, ink, and accent (terracotta). [cite_start]The accent appears on exactly one primary interactive element per screen state[cite: 252].

03 — Typography

Typography is the most critical system in Tippani. [cite_start]The app exists to serve writing[cite: 268].

3.1 The Three Fonts

  • Vesper Libre (Latin Display): High-contrast serif with ink-traps. [cite_start]Used for large headings and app identity[cite: 271].
  • Hind (Interface & Devanagari): Humanist sans-serif designed for bilingual use. [cite_start]Primary font for UI and body[cite: 275].
    • [cite_start]
  • DM Mono (Monospace): Used for timestamps, counts, and technical data[cite: 280].

3.2 Devanagari Rules

Critical: NEVER apply letter-spacing to Devanagari text. [cite_start]It breaks the matra ligatures[cite: 288]. [cite_start]Line-height for Hindi body text must be 1.8–2.0[cite: 287].

05 — Border Radius (Pill System)

15 — Do's and Don'ts

Always Do

    [cite_start]
  • Use --r-pill for ALL primary interactive elements[cite: 670].
  • Display Hindi labels first. [cite_start]This is a Hindi-first product[cite: 672].
    • [cite_start]
  • Show key-press shadow collapse on every tap[cite: 671].
    • [cite_start]
  • Use hard offset shadows (zero blur radius)[cite: 676].

Never Do

    [cite_start]
  • DON'T use pure black (#000000) or white (#FFFFFF)[cite: 685].
    • [cite_start]
  • DON'T apply letter-spacing to Devanagari text[cite: 686].
    • [cite_start]
  • DON'T use soft Gaussian blurs for shadows[cite: 687].
    • [cite_start]
  • DON'T use decorative accent colors[cite: 689].
